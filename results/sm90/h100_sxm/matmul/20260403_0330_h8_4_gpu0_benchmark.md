## Implementation Details

### CuTe DSL (WGMMA) — `cute_gemm_sm90.py`
- WGMMA 128×256×64 tile, 2 warpgroups (atom_layout 2×1×1)
- All-cooperative (no dedicated DMA warp)
- Multi-stage TMA pipeline, epilogue SMEM reuses mainloop sA
- Persistent tile scheduler with CTA swizzle (group_size_m=8)

### Generated (CUTLASS C++) — `generated.cu`
- CollectiveBuilder with KernelTmaWarpSpecializedCooperative
- Big: 128×256×64, Small: 128×128×64
- ptxas warning: wgmma pipeline crossing function boundary

### cuBLAS — `torch.mm()` BF16
- sm90_xmma_gemm kernel, 128×256×64 tile

## Environment Notes

- `cuda.bindings` 12.8.0 ships `cuda.bindings.runtime` v13.2 (libcudart.so.13)
- CUDA 13.2 runtime requires driver > 12.8 compat → `cudaGetDeviceCount` fails
- Workaround: monkey-patch `cuda.bindings.runtime.cudaGetDeviceCount` to use driver API `cuDeviceGetCount`
- `LD_LIBRARY_PATH=/usr/local/cuda-12.8/compat:...` required for CuTe DSL

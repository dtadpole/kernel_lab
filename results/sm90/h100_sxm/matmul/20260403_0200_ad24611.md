## Implementation Details

### CuTe DSL (WGMMA) — `data/fixtures/sm90/matmul/cute_gemm_sm90.py`

Official CUTLASS 4.3.5 Hopper CuTe DSL example, adapted for BF16:
- **MMA**: WGMMA (warpgroup-level, 64×256×16 instruction)
- **Tile**: 128×256×64
- **Warpgroups**: 2 MMA (atom_layout 2×1×1), all-cooperative (no dedicated DMA)
- **Pipeline**: Multi-stage TMA G2S with PipelineTmaAsync
- **Epilogue**: TMA S2G store with PipelineTmaStore
- **Scheduling**: Persistent tile scheduler

### Generated (CUTLASS C++) — `data/generated/sm90/matmul/generated.cu`

CUTLASS 3.x CollectiveBuilder API:
- **Schedule**: `KernelTmaWarpSpecializedCooperative`
- **Tile**: 128×256×64 (big) / 128×128×64 (small)
- **Auto stage count** with epilogue SMEM carveout
- **Persistent scheduler**
- ptxas warning: `wgmma pipeline crossing function boundary` — causes 5-7% gap vs CuTe DSL

### cuBLAS — `torch.mm()` BF16

- Kernel: `sm90_xmma_gemm_bf16bf16_bf16f32_f32_nn_n_tilesize128x256x64_warpgroupsize2x1x1`
- Same tile/warpgroup architecture as CuTe DSL and Generated

## Key Findings

1. **CuTe DSL WGMMA beats cuBLAS at all sizes** (105-136%), peaking at 755 TFLOPS for 8192
2. **Generated (CUTLASS C++) achieves 97% cuBLAS at 8192** — limited by ptxas function boundary serialization
3. **CuTe DSL is 13% faster than Generated at 8192** — MLIR JIT compiler produces better instruction scheduling than nvcc/ptxas
4. **Per-warp mma.sync (SM120 style) achieves only 40-50% of cuBLAS on H100** — WGMMA is mandatory for SM90

## Environment Notes

- H100 driver 550.90.07 (CUDA 12.4) requires `cuda-compat-12-8` package for CuTe DSL
- Set `LD_LIBRARY_PATH=/usr/local/cuda-12.8/compat:$LD_LIBRARY_PATH` and `CUTE_DSL_ARCH=sm_90a`
- CuTe DSL venv: `cuda-bindings==12.8.0` + `nvidia-cutlass-dsl==4.4.2` + `torch==2.6.0+cu124`

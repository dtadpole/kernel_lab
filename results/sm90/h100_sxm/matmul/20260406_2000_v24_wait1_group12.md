# SM90 Matmul — v24: wgmma_wait1 + GROUP_M=12

**Date**: 2026-04-06 20:00
**Host**: devvm8491 (h8_4), GPU 4, NVIDIA H100 SXM 96GB
**CUDA**: 13.2, Driver 580.65.06
**Harness**: eval_harness (5 warmup + 10-20 timed runs)

## Architecture

- 3 warpgroups (384 threads): 1 producer + 2 consumers
- CTA tile: 128×256, TILE_K=64, 4-stage TMA pipeline
- WGMMA m64n64k16 with trans-b=1 (4 quarters × 4 k-steps per stage)
- 168 registers, 0 spills, 0 stack frame, 1 barrier

## Key Changes (v24 vs v23)

1. **wgmma_wait2 → wgmma_wait1** (+12.0% at 8192): wait1 signals mbar_empty
   one stage earlier, allowing the producer to reload sooner. wait2 delayed
   the release, causing producer stalls that dominated at large K.

2. **GROUP_M=16 → GROUP_M=12** (marginal +0.3%): Slightly better L2 B-tile
   reuse for square matrices. Not significant on its own.

## Root Cause of v23 Regression

wgmma_wait2 was intended to allow 2 WGMMA groups in-flight for better
compute pipelining. However, it delayed mbar_empty signaling by 1 stage,
causing the producer to stall longer before it could reload pipeline stages.
At large K (128 stages for 8192), this producer stall dominated, reducing
effective throughput by 12%.

## Remaining Gap to cuBLAS (~3%)

cuBLAS achieves ~760 TF via:
1. **Cluster 2×1 with TMA multicast** — halves A memory traffic
2. **TMA store epilogue** — bulk SMEM→GMEM instead of per-thread stores
3. **NVCC-internal optimizations** — WGMMA pipelining across function boundaries

These require significant architectural changes beyond parameter tuning.

## Failed Attempt: Descriptor Optimization (v25)

Deriving B descriptors from db0 via desc_advance (saving 3 make_desc calls)
and removing epilogue boundary checks showed no measurable improvement.
ALU overhead reduction was too small (~0.1%) to affect performance.

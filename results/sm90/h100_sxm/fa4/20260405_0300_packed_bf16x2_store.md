# FA4 Packed BF16x2 O Store — 454–599 TFLOPS (+3–7%)

## Hardware

- **GPU:** NVIDIA H100 SXM5 96GB HBM3
- **Architecture:** SM 9.0a (Hopper)
- **Peak BF16:** 800 TFLOPS (650W TDP)
- **Host:** devvm8490 (h8_3), GPU 7, CUDA 12.9

## Objective

Reduce O epilogue store overhead by packing adjacent BF16 values into 32-bit
stores. The O store uses the WGMMA accumulator layout where s2={0,1} map to
adjacent columns — perfect for BF16x2 packing via `cvt.rn.bf16x2.f32`.

## What Changed

**Before:** Each thread stores 64 individual BF16 values (2 bytes each) to
global memory via 64 scalar STG.16 instructions. The stores are uncoalesced
(strided by `seq_stride = H*D*2 = 4096` bytes between rows).

**After:** Pack s2=0 and s2=1 values into BF16x2 (32-bit) using the existing
`pack_bf16(f32, f32)` helper, then store via STG.32. This halves the store
instruction count from 64 to 32 per thread (16 for O_lo, 16 for O_hi).

```c
// Before: 2 × STG.16 per (p4, s2) pair
my_O_base[row * seq_stride + col] = __float2bfloat16(O_lo_acc[idx]);

// After: 1 × STG.32 per p4 (both s2 values packed)
uint32_t lo_packed = pack_bf16(O_lo_acc[idx0], O_lo_acc[idx1]);
*(uint32_t*)&my_O_base[row_offset + col_base] = lo_packed;
```

## Why It Helps

1. **Halved store instruction count**: 32 STG.32 instead of 64 STG.16
2. **Better memory coalescing**: 32-bit stores have wider coverage per
   transaction, reducing the number of L2 cache line fills
3. **More impactful on small configs**: O store is fixed overhead per Q block.
   With fewer KV iterations (small S), the O store is a larger fraction of
   total time

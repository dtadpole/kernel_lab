# Formal Benchmark

**The sole official benchmark.** All performance results, improvement decisions,
and gem records come exclusively from `ik:bench`. Results from `ik:exec` or
`ik:optimize` are preliminary and never authoritative.

## Arguments

| Arg | Required | Default | Description |
|-----|----------|---------|-------------|
| `$0` (kernel) | **yes** | — | Kernel name: `fa4`, `matmul`, etc. |
| `gpu` | no | from host config | GPU index (`CUDA_VISIBLE_DEVICES`) |
| `arch` | no | auto-detect | GPU arch: `sm90`, `sm120` |
| `impls` | no | `all` | Impl slugs to bench, or `all` |
| `timeout` | no | `300` | Per-config timeout (seconds) |

GPU is **session-sticky**: once set, all subsequent ik invocations reuse it.

## Usage

```bash
cd /home/zhenc/kernel_lab

# Only kernel is required — everything else auto-detected
.venv/bin/python -m cuda_exec.formal bench.kernel=fa4
.venv/bin/python -m cuda_exec.formal bench.kernel=matmul

# Overrides
.venv/bin/python -m cuda_exec.formal bench.kernel=fa4 bench.gpu=2
.venv/bin/python -m cuda_exec.formal bench.kernel=fa4 bench.arch=sm90
.venv/bin/python -m cuda_exec.formal bench.kernel=fa4 'bench.impls=[ref-cudnn,peak-cuda]'
```

## What It Does

1. Snapshots sources + configs to `~/kernel_lab_kb`
2. Discovers all impl slugs via `list_impls()` (see `artifacts/slug-resolution.md`)
3. Measures `.py` impls via Python harness, compiles + trials `.cu` impls via C harness
4. Writes `results.json` to `kernel_lab_kb`; creates a **gem** if any config improves

## Output Format

The table is auto-generated by `format_results_table()` and printed to stdout.
**Always output this table — never skip it.**

```
**NVIDIA H100 SXM R&R SKU 96GB** | host: h8_4 | driver: 595.45.04 | CUDA: 13.2 | peak: 800.0 TFLOPS (BF16 TC)

| Config        | ref-cublas     | ref-pytorch            |
|               |                | PyTorch 2.11.0+cu130   |
|               | ms    TFLOPS   | ms    TFLOPS   speedup |
|---------------|----------------|------------------------|
| mat-256x256   |  0.012     2.9 |  0.032     1.1 ✓ 0.37x |
| mat-4096x4096 |  0.198   692.8 |  0.203   675.8 ✓ 0.98x |
| mat-8192x8192 |   1.45   760.8 |   1.46   754.2 ✓ 0.99x |
|---------------|----------------|------------------------|
| % of peak     |          95.1% |                  94.3% |

Golden: ref-cublas — ✓/✗ = correctness vs golden
```

**Table structure:**
- **Env line**: GPU model, host slug, driver, CUDA version, peak TFLOPS
- **Header row 1**: impl slugs
- **Header row 2**: version labels (cuDNN, PyTorch, FA4 — auto-detected)
- **Header row 3**: `ms    TFLOPS   speedup`
- **Data rows**: per-config latency, TFLOPS, ✓/✗ correctness, speedup (2 decimals)
- **% of peak**: best TFLOPS / GPU peak
- **Footer**: golden identity, source paths

**Speedup** uses 2 decimal places (`1.07x`, `0.97x`) for 1% precision.
Golden column has no ✓/✗ or speedup (it IS the baseline).

## Rules

- **ALL configs** trialed, no exceptions
- **At least one ref-* impl** required
- **No profiling** — bench is measurement only
- **Snapshot-first** — never evaluates original files directly
- **Read-only** — bench does not modify source files

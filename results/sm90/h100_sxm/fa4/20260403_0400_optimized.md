## What Changed (vs `20260403_0300_initial`)

### Generated kernel optimizations

1. **Softmax division → MUFU.RCP**: Replaced `O /= rowsumexp` (full-precision fdiv → FCHK + slowpath CALL) with inline PTX `rcp.approx.ftz.f32`. Eliminates 128 FCHK + 10 CALL + 104 slowpath instructions.

2. **exp2f → inline PTX**: Replaced `exp2f()` with inline PTX `ex2.approx.ftz.f32`. Eliminates 128 FCHK guards the compiler inserts before MUFU.EX2.

3. **BLOCK_Q 128→64**: Halves WARP_Q from 32 to 16, reducing O_rmem from [2][16][4] to [1][16][4]. Cuts live register pressure by ~64 floats. Doubles grid blocks for better SM occupancy.

4. **V double-buffered**: V shared memory is now double-buffered (like K), overlapping V load with PV compute. SMEM: 80KB → 80KB (Q halved from 32→16KB offsets the V doubling).

5. **`__launch_bounds__(160, 2)`**: Targets 2 blocks/SM occupancy (up from 1).

## Analysis

- **Generated kernel: ~2x improvement** (100–115 → 198–231 TFLOPS)
- **FA4 CuTe DSL**: 541–647 TFLOPS (unchanged)
- **cuDNN**: 553–658 TFLOPS (unchanged)
- **Generated vs FA4/cuDNN**: ~35% of reference performance (up from ~17%)
- **H100 BF16 utilization**: Generated at 20–23% of peak 989.5 TFLOPS

### Remaining gap

The generated kernel uses SM80-compatible `mma.sync.m16n8k16` which executes synchronously — scalar softmax work and tensor core MMA cannot overlap within a warp. FA4/cuDNN use SM90-native `wgmma` (async warp group MMA) + TMA + mbarrier, enabling full compute/load/softmax overlap. Closing the remaining 3x gap requires migrating to the SM90-native instruction set.

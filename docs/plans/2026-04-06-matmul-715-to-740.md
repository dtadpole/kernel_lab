## Step 2: Descriptor Hoisting (target: mio_throttle 11.7% partial reduction)

**What**: Pre-compute per-stage base descriptors outside K-loop. Inside the loop,
only update the address field via `desc_advance`.

**Why**: SASS shows UMOV of constant 0x40000040 (LBO+SBO+swizzle bits) between every
WGMMA pair. These are redundant â€” the upper bits never change within a stage.

**How**:
1. Before K-loop: build base descriptors for each of the 4 stages (A + 4Ã—B = 5 descs Ã— 4 stages = 20 descriptors)
2. Store in local variables (compiler puts in registers)
3. K-loop: index by `kt % STAGES`, advance for k-substep only
4. This reduces `make_desc` calls from 5/iteration to 0/iteration (just lookups)

**Expected gain**: Small. The compiler may already optimize this, and the UMOV constants
might be from WGMMA's internal descriptor processing, not our code.

**Risk**: Low. Pure refactoring, no correctness impact.

## Step 3: Thread Block Clusters + TMA Multicast (target: long_scoreboard 21.6% reduction)

**What**: Launch with `__cluster_dims__(2, 1, 1)`. Two SMs in a cluster share B tiles
via TMA multicast. Each SM loads its own A tile, but B is loaded once and broadcast.

**Why**: B accounts for 67% of TMA traffic (32KB out of 48KB per stage).
Multicast halves B DRAM reads: DRAM for B drops from ~0.87GB to ~0.44GB.

**How**:
1. Add `__cluster_dims__(2, 1, 1)` to kernel signature
2. Use `cudaLaunchKernelEx` with cluster config (or `__cluster_dims__` attribute)
3. TMA load B with multicast mask: `mcast_mask = 0x3` (both SMs in cluster)
4. TMA load A without multicast (each SM needs different A rows)
5. Adjust CTA swizzle: pairs of M-tiles in the same cluster share B
